# 🚀 ExamX Setup Guide

Follow these steps carefully before starting your assessment.

---

## Install ExamX Extension

1. Open **Visual Studio Code**
2. Go to Extensions (`Ctrl + Shift + X`)
3. Search for **ExamX**
4. Click **Install**

## Login and Start the Exam

After installing the ExamX extension:

- A **Login** button will appear in the **bottom-left corner** of VS Code.
- Click the **Login** button to begin the authentication process.
- Enter your registered email address.
- Complete the email verification.
- Enter the **Exam Passcode** provided by the examiner.

Once verification is successful:

- The assigned exam will load automatically.
- Click **Start Exam** to begin.

After the exam starts:

- A **timer** will appear in the bottom-left corner.
- Monitoring and activity logging will begin.
- A **Submit Exam** button will be available for manual submission.

You may submit the exam manually using the **Submit Exam** button, or it will be automatically submitted when the timer expires.


# 🛡️ Technical Assessment – Rules & Guidelines

Please read the following carefully. These rules are mandatory.

# ⏱️ Commit & Version Control Rules (MANDATORY)

Your Git commit history is part of the evaluation.

## You MUST:

- Commit your code **at least every 15 minutes**
- Use **meaningful commit messages**
- Keep commits small and logical
- Maintain a clean and readable history

## You MUST NOT:

- Squash commits
- Create one large final commit
- Push the code (pushing is NOT required)

### ✅ Example Good Commit Messages

```
feat: implement add to cart functionality
fix: prevent state mutation in cart reducer
refactor: extract discount calculation logic
fix: handle coupon validation edge case
```

⚠️ Your commit history will be reviewed carefully.


# 🤖 AI Usage Policy

- Do NOT copy code from AI tools or online sources.
- You must fully understand every line of code you write.
- You may be asked to explain your implementation live.

Unrealistic implementation speed or suspicious commit patterns may lead to disqualification.


# 🖥️ Project & Environment Rules

You must follow these constraints strictly:

- Use the provided **React + TypeScript** setup.
- Do NOT install additional state management libraries.
- Do NOT modify any files outside the `src` folder.
- Do NOT change the project structure.
- Do NOT rename or restructure configuration files.

# 🧩 System Requirements

Before starting, ensure you have:

- Visual Studio Code **v1.80 or later**
- Stable internet connection
- Access to your registered email (for verification)

- Attempting to bypass monitoring

- Opening multiple VS Code instances for the same exam
- Tampering with tracking or logs
